> **日期**：2026-03-17

> **实验类型**：评估协议修正后的 v2 baseline 全量重跑（不覆盖 legacy `phase1_v2_*`）

## 1. 任务目标

1. 将 continuous validation / test 的评估采样改为 `SequentialFixedWindowSampler`
2. 将 continuous `fp-bps / R²` 改为全局累计版
3. 将 trial-aligned 主 PSTH 指标统一为 `per_neuron_psth_r2`
4. 在不改变 v2 baseline 训练超参的前提下，重跑 `1.3.4` 的 6 个条件
5. 对比 legacy `phase1_v2_*` 与新 `phase1_v2_evalfix_*` 的 valid / test 结果

## 2. 代码改动摘要

- `examples/neurohorizon/train.py`
  - continuous valid / test 改为 `SequentialFixedWindowSampler`
  - 新增 epoch 级全局累计 `fp-bps / R²` 聚合
  - 新增 `test_dataloader()`，continuous / trial-aligned 均支持
- `torch_brain/utils/neurohorizon_metrics.py`
  - 新增 `fp_bps_stats / r2_stats / fp_bps_per_bin_stats` 及 finalize helpers
- `scripts/analysis/neurohorizon/eval_phase1_v2.py`
  - 新增 `--split {valid,test}`
  - continuous 部分改为 sequential + 全局累计
  - trial-aligned 主指标改为 `per_neuron_psth_r2`
- `scripts/analysis/neurohorizon/eval_psth.py`
  - 指标输出统一为 `per_neuron_psth_r2`
- `scripts/analysis/neurohorizon/phase1_v2_visualize.py`
  - 改为显式 `--protocol legacy|evalfix --split valid|test`，避免 legacy / evalfix 图表混写
  - 兼容旧 `psth_r2` 和新 `per_neuron_psth_r2` JSON key
- 新增：
  - `scripts/analysis/neurohorizon/run_phase1_v2_evalfix.sh`
  - `scripts/analysis/neurohorizon/compare_phase1_v2_evalfix.py`

- `06` 使用当前 benchmark latest 250ms reference：

**关键图表结论**：
1. `01_fpbps_vs_window.png`：continuous 仍保持 `0.2164 -> 0.1823 -> 0.1374` 的缓慢衰减，evalfix 没有改变 `250ms > 500ms > 1000ms` 的主排序。
2. `02_perbin_fpbps_decay.png`：`250ms-cont` 全 12 个 bin 都保持正值且相对平稳（`0.1735–0.2690`）；`500/1000ms-cont` 的后段分别衰减到 `0.1206 / 0.0915`。
3. `03_perbin_fpbps_heatmap.png`：三种窗口的 horizon 衰减被压到同一张 heatmap 里，直观看出 `1000ms` 的掉速主要来自远端 bins。
4. `04_continuous_metric_summary.png`：continuous `fp-bps` 与 `R²` 两条曲线都随窗口增大而下降，其中 `fp-bps` 的下降更明显。
5. `05_training_curves.png`：continuous 三组 best validation `fp-bps` 分别出现在 epoch `159 / 259 / 269`，之后只小幅回落，说明主线 baseline 收敛稳定。
6. `06_benchmark_comparison.png`：当前 250ms reference 中，`NeuroHorizon = 0.2223` 仍高于 `IBL e300 = 0.1938`，而 `Neuroformer faithful canonical 250ms rollout = -8.0350` 仍显著为负。

## 7. 结论

1. continuous baseline 在 evalfix 协议下保持稳定，三种窗口的 valid `fp-bps` 都略高于 legacy 结果
2. trial-aligned 在 300 epochs 下仍然显著落后于 continuous；500/1000ms 的 trial `fp-bps` 继续为负
3. `per_neuron_psth_r2` 统一后，旧的 population-mean `PSTH-R2` 结论不应继续直接沿用
4. 本轮重跑确认：评估协议修正没有推翻 v2 baseline 的主排序，但显著收紧了指标口径和结果解释边界
