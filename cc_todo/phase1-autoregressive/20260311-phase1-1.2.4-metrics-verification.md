## 验证目标

验证 1.1.7（评估指标）和 1.1.9（Trial-Aligned Sampler）的实现正确性。

## 重要说明：合成数据验证

本验证脚本使用**合成数据**验证 fp-bps 公式的数学正确性，**不是**使用真实训练模型的推理结果。

- **null_log_rates**：从合成 target（Poisson 采样）中统计 per-neuron 均值得到，与 NLB 参考实现 `np.nanmean(spikes, ...)` 方式一致
- **目的**：验证公式实现与 NLB 参考代码（`nlb_tools/evaluation.py`）数值一致，而非评估模型性能
- **局限**：实际训练评估中，null_log_rates 应由 `compute_null_rates()` 从**训练集**统计得到（更严谨，不偷看测试数据）；完整的模型推理 + 真实 null rates 的验证在 1.3.4 实验中执行

## 修复记录

### 第一次修复（PyTorch + NaN）
初次运行时 2 项失败：
1. **Test 3b**：PyTorch 2.6 默认 `weights_only=True`，需加 `weights_only=False` → 已修复
2. **Test 5**：HDF5 中 `target_id` 为 float64 类型，部分 trial 含 NaN（无效 trial），需 `np.isnan()` 过滤后检查范围 → 已修复

### 第二次修复（合成数据参数 + NLB 交叉验证）
初版测试使用了极端的合成参数，导致 fp-bps 数值远超真实范围：
- **旧 Test 2**：`null_log_rates=0` + `randn*3.0`（极端随机），fp-bps=-39.56（实际应 ~-0.5）
- **旧 Test 3**：`null_log_rates=5.0`（rate=exp(5)≈148，荒谬的 null model），fp-bps=177.49（实际应 ~0.05-0.5）

**问题原因**：旧测试只关注符号正确（正/负/零），未关注数值量级是否与 NLB 论文一致（~0.2）。

**修复方案**：
- 改用时间变化的合成 rates（base + time_modulation + trial_variation），模拟真实神经活动
- null model 从合成数据的 per-neuron 均值统计（与 NLB 方式一致）
- 新增 Test 3a：直接与 NLB 参考实现交叉验证，确认数值一致
