# Phase 1.3.4 v2 实验：fp-bps 主指标 + 连续/trial-aligned 对比

**日期**：2026-03-12
**任务编号**：1.3.4
**状态**：完成

## 实验目的

1. 用 fp-bps 替代 R-squared 作为主要评估指标，重新进行预测窗口实验
2. 对比连续训练 vs trial-aligned 训练两种数据组织方式的效果
3. 计算 PSTH-R-squared 评估群体水平预测质量
4. 分析 per-bin fp-bps 衰减趋势

## 实验配置

6 个训练运行（3 窗口 x 2 模式），全部 300 epochs，seed=42：

| 条件 | 模式 | pred_window | bins | batch | config |
|------|------|-------------|------|-------|--------|
| 250ms-cont | 连续 | 250ms | 12 | 64 | train_v2_250ms.yaml |
| 250ms-trial | trial-aligned | 250ms | 12 | 64 | train_v2_250ms_trial.yaml |
| 500ms-cont | 连续 | 500ms | 25 | 64 | train_v2_500ms.yaml |
| 500ms-trial | trial-aligned | 500ms | 25 | 64 | train_v2_500ms_trial.yaml |
| 1000ms-cont | 连续 | 1000ms | 50 | 32 | train_v2_1000ms.yaml |
| 1000ms-trial | trial-aligned | 1000ms | 50 | 32 | train_v2_1000ms_trial.yaml |

## 关键发现

### 1. 连续训练稳定有效
- fp-bps 随窗口增长缓慢衰减：0.212 -> 0.174 -> 0.132（-18% -> -24%）
- R-squared 同样缓慢衰减：0.261 -> 0.237 -> 0.229
- 所有连续模型 fp-bps 均为正值（优于 null model）

### 2. Trial-aligned 训练严重过拟合
- 所有 3 个 trial-aligned 模型在 300 epochs 后均出现负 fp-bps（-0.17 到 -0.25）
- 根因：trial-aligned 模式样本量极少（约 186 trials vs 连续模式 6372 windows）
- 训练曲线显示 trial 模型在早期（约 30-50 epochs）达到最佳后开始过拟合
- 建议后续使用 early stopping 或减少 epochs

### 3. PSTH-R-squared 有趣发现
- Trial-aligned 模型虽然 fp-bps 负值，但 PSTH-R-squared 反而更高（0.60-0.80 vs 0.15-0.68）
- 解释：PSTH 是 population-level 指标，跨 trial 平均后消除了单次预测的误差
- Trial-aligned 模型学到了 direction-dependent 群体模式，但 single-trial 预测已过拟合

### 4. Per-bin fp-bps 衰减分析
- 连续模型 250ms：12 个 bin 的 fp-bps 相对稳定（0.16-0.24），无明显衰减
- 连续模型 500ms/1000ms：中后段 bin 有轻微衰减趋势
