# Phase 1.3.5 IBL-MtM 风格 bps 指标扩展

## 1. 任务目标

在不改变当前 `evalfix` 主评估协议的前提下，为 `1.3.4` 的 continuous baseline 补充一个更接近 IBL-MtM 原始表格口径的对照指标：

- 主指标：`fp-bps`（global spike-weighted + train-split null）
- 新指标：`ibl_mtm_bps`（per-neuron mean + eval-split null）

本任务仅覆盖 `250ms / 500ms / 1000ms` 三个 continuous 条件，不重新训练模型。

## 2. 代码范围

- `torch_brain/utils/neurohorizon_metrics.py`
- `scripts/analysis/neurohorizon/eval_phase1_v2.py`
- `scripts/analysis/neurohorizon/compare_phase1_v2_ibl_metric.py`

## 3. 目标目录

- causal evalfix checkpoints:
  - `results/logs/phase1_v2_evalfix_250ms_cont/`
  - `results/logs/phase1_v2_evalfix_500ms_cont/`
  - `results/logs/phase1_v2_evalfix_1000ms_cont/`
- comparison:
  - `results/logs/phase1_v2_metric_extension_comparison/`

## 4. 评估命令

```bash
conda activate poyo

python scripts/analysis/neurohorizon/compare_phase1_v2_ibl_metric.py
```

## 5. 待回填结果

### 5.1 valid/test 结果

| 条件 | valid fp-bps | valid ibl_mtm_bps | test fp-bps | test ibl_mtm_bps |
|------|--------------|-------------------|-------------|------------------|
| 250ms-cont | 0.2164 | 0.2234 | 0.2223 | 0.2321 |
| 500ms-cont | 0.1823 | 0.1609 | 0.1740 | 0.1555 |
| 1000ms-cont | 0.1374 | 0.0381 | 0.1348 | 0.0532 |

### 5.2 结论

1. 两种指标都保持 `250ms > 500ms > 1000ms` 的窗口排序
2. `ibl_mtm_bps` 在 `1000ms` 上下降更剧烈，显示该口径对长窗口下的难预测 neuron 更敏感
3. `ibl_mtm_bps` 应作为 comparison metric 保留，不替代主 `fp-bps`
