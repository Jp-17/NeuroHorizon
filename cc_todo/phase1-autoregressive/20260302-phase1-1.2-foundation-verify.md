## Supplementary Analysis: Teacher Forcing vs Autoregressive Deep Dive

### Concept Definitions

#### Teacher Forcing (TF)
- **Definition**: In autoregressive model training, each time step uses ground truth (not model's own prediction) as input
- **Standard scenario (e.g., GPT training)**: step t's input = embedding(GT_token_{t-1})
- **Purpose**: Eliminate error accumulation, stabilize gradients, accelerate convergence
- **Cost**: During training the model only sees correct history (GT); during inference it must use its own potentially incorrect predictions -> this distribution mismatch is called **exposure bias**

#### Autoregressive Inference (AR)
- **Definition**: During inference with no GT available, model must use its own step t-1 prediction as step t's input
- **Error accumulation**: step t prediction error -> becomes step t+1 input -> errors compound
- **Quantification**: For T-step prediction, error upper bound may grow linearly or exponentially with T
- **Classic mitigations**:
  - Scheduled Sampling (Bengio et al., 2015): gradually increase probability of using predictions instead of GT during training
  - Various Exposure Bias Reduction variants

### Why TF = AR in NeuroHorizon

#### Code-Level Trace

Current decoder initialization in neurohorizon.py:
```python
bin_queries = self.bin_emb[:, :T_pred, :].expand(B, -1, -1).clone()
```

`bin_emb` is `nn.Parameter(shape=[1, max_pred_bins, dim])`, a fixed learnable embedding.
It does NOT depend on: (1) input data (2) encoder latents (3) any previous step predictions

#### Key Difference from GPT

| Dimension | GPT | NeuroHorizon Decoder |
|-----------|-----|---------------------|
| Input query | embedding(token_{t-1}), **depends on previous output** | bin_emb[t], **fixed parameter** |
| Causal mask effect | Prevents seeing future token content | Prevents seeing future fixed embeddings (meaningless) |
| TF vs AR | **Different** (exposure bias exists) | **Identical** (no exposure bias) |

GPT: input_t = Embedding(output_{t-1}) -> previous step output directly determines next input
NeuroHorizon: input_t = bin_emb[t] -> positional parameter, independent of output

#### Mathematical Proof

Let r_t^{TF} be bin t's representation in forward(), r_t^{AR} in generate().

forward(): All T bins processed in parallel, causal mask lets bin t see only bin 0..t-1.
generate(): Sequential processing of bin 0, 1, ..., T-1.

For bin t:
- TF: r_t^{TF} = f(bin_emb[t], latents, {bin_emb[0..t-1] + g(latents)})
- AR: r_t^{AR} = f(bin_emb[t], latents, {bin_emb[0..t-1] + g(latents)})

Both have identical inputs -> r_t^{TF} = r_t^{AR}

#### Experimental Verification
- ar_verify.py result: max_diff = 3e-6 (floating point precision level)
- AR/TF R2 ratio = 1.0000

### Three Approaches to Implement True AR

#### Approach A: Query Augmentation
- Add encoded previous prediction to current bin query: `bin_query[t] = bin_emb[t] + encode(predicted_rates[t-1])`
- TF: use GT counts (shift-right), parallel computation
- AR: use model predictions, step-by-step
- Pros: minimal code change, fully parallel training, causal mask immediately meaningful
- Cons: feedback compressed to single vector

#### Approach B: Prediction Memory Cross-Attention (Perceiver AR inspired)
- Add second cross-attention in decoder to prediction memory sequence
- TF: memory = encode(GT counts) with causal cross-attn mask
- AR: memory accumulates step-by-step
- Pros: richer information flow (dedicated cross-attn), separates 'history' and 'prediction history'
- Cons: new cross-attention parameters, moderate code change

#### Approach C: Perceiver AR Full Adaptation (Re-encode with Predictions)
- Convert predictions to spike tokens, append to encoder input, re-encode each step
- Pros: most complete information fusion, most aligned with POYO/Perceiver native design
- Cons: re-runs encoder every step (compute x T_pred), largest architecture change

### Perceiver IO -> Perceiver AR Evolution
- Perceiver IO (Jaegle et al., 2021): output queries cross-attend to latents, essentially a parallel decoder
- Perceiver AR (Hawthorne et al., 2022): each latent corresponds to an output position + causal cross-attention
- **For NeuroHorizon**: current architecture resembles Perceiver IO's parallel decoder; borrowing Perceiver AR's prediction feedback mechanism would make it truly AR

### Positioning
- Current model: **causal forward predictor** (predicts from observation to prediction window)
- NOT a true **autoregressive generator** (no prediction information flow between bins)
- AR fix (Approach A/B/C) will make it truly AR
- Pre-fix experimental results (1.2.1/1.2.2/1.3) still have reference value, but AR vs non-AR comparison is meaningless
