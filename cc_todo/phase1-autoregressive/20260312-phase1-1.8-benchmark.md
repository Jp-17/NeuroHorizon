## 2026-03-12 | 1.8.2 环境配置与模型部署

**完成内容**：

1. **conda envs_dirs 映射**：配置 `neural-benchmark/envs/` 为 conda 环境存储目录
   ```bash
   conda config --prepend envs_dirs /root/autodl-tmp/NeuroHorizon/neural-benchmark/envs
   ```

2. **Clone 模型仓库**（均在 `neural-benchmark/benchmark_models/` 下）：
   - `ndt2/` ← https://github.com/joel99/context_general_bci
   - `neuroformer/` ← https://github.com/a-antoniades/Neuroformer
   - `ibl-mtm/` ← https://github.com/colehurwitz/IBL_MtM_model

3. **创建共用 conda 环境 `benchmark-env`**：
   - Python 3.10.19
   - PyTorch 2.2.1+cu118（服务器 CUDA 13.0 driver 兼容）
   - lightning 2.2.0 + pytorch_lightning 2.2.0
   - 共用依赖：numpy<2, scipy, pandas, matplotlib, seaborn, scikit-learn, h5py, einops, wandb, hydra-core, omegaconf, tqdm
   - NDT2 特有：pynwb, mat73, dacite, gdown, ordered-enum, yacs
   - IBL-MtM/Neuroformer 共有：timm, transformers, accelerate, datasets, torcheval
   - 环境位置：`/root/autodl-tmp/NeuroHorizon/neural-benchmark/envs/benchmark-env/`

4. **兼容性补丁**：
   - NDT2 `model.py` 第 1382-1384 行：修复 `pytorch_lightning.plugins.precision.native_amp` 在 lightning 2.2 中不存在的问题
     - 改用 try/except 回退到 `lightning.pytorch` 命名空间
     - 仅修改 import 语句，模型架构和 forward pass 完全不变

5. **Import 验证结果**：
   - ✅ NDT2: `from context_general_bci.model import BrainBertInterface` 成功
   - ✅ Neuroformer: `from neuroformer.model_neuroformer import Neuroformer` 成功（需 PYTHONPATH=neuroformer:.）
   - ✅ IBL-MtM: `from models.ndt1 import NDT1` 成功

6. **.gitignore 更新**：添加 `neural-benchmark/envs/` 和 `neural-benchmark/benchmark_models/`

**遇到的问题与解决**：
- **PyTorch 版本冲突**：NDT2 要求 1.13.1+cu117，IBL-MtM 要求 2.2.1+cu118，服务器 CUDA 13.0 不支持 cu117
  - 解决：统一使用 PyTorch 2.2.1+cu118，对 NDT2 做最小兼容补丁
- **pytorch_lightning API 变更**：`native_amp` 模块在 lightning 2.x 中移除
  - 解决：在 NDT2 model.py 中添加 try/except 回退 import
- **NumPy 2.x 兼容性**：PyTorch 2.2.1 不兼容 NumPy 2.x
  - 解决：pin `numpy<2`
