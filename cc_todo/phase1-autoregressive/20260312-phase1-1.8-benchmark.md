## 2026-03-12 | 1.8.3 模型适配与对比实验

### Part A: 适配层开发

**完成内容**：

1. **`neural-benchmark/adapters/base_adapter.py`**（约 250 行）：
   - `BenchmarkConfig` 数据类：bin_size=20ms, obs/pred window, epochs, lr 等
   - `bin_spike_events()`: spike events → 20ms binned counts（向量化 np.add.at）
   - `create_tb_dataset()`: 创建 torch_brain Dataset（root + config + split）
   - `BenchmarkDataset`: 包装 torch_brain Dataset，slide window 切片，返回 spike_counts/masks/unit_ids
   - `compute_benchmark_null_rates()`: 从训练集计算 per-unit null log rates
   - `evaluate_model()`: 统一评估函数（fp-bps, R², per-bin fp-bps, Poisson NLL）

2. **`neural-benchmark/adapters/ndt2_adapter.py`**（约 145 行）：
   - `NDT2Wrapper`: MAE-style encoder, 4.8M 参数
   - 特点：双向注意力 + 预测窗口输入置零（mask future bins）
   - hidden_size=256, n_heads=4, n_layers=6, dropout=0.2
   - 正弦位置编码, Linear → LayerNorm → GELU → Linear output head

3. **`neural-benchmark/adapters/ibl_mtm_adapter.py`**（约 160 行）：
   - `IBLMtMWrapper`: NDT1-style causal encoder, 10.7M 参数
   - 特点：causal mask + 预测窗口输入置零，Softsign embedding
   - hidden_size=512, n_heads=8, n_layers=5, dropout=0.4, embed_mult=2
   - 训练 loss: 70% prediction window + 30% shifted obs auxiliary

4. **`neural-benchmark/adapters/neuroformer_adapter.py`**（约 150 行）：
   - `NeuroformerWrapper`: GPT-style causal decoder, 4.9M 参数
   - 特点：causal mask + 预测窗口输入置零，2-layer input embedding
   - hidden_size=256, n_heads=8, n_layers=6, dropout=0.35
   - 训练 loss: 70% prediction window + 30% shifted obs auxiliary

5. **`neural-benchmark/benchmark_train.py`**（约 200 行）：
   - 统一训练脚本，CLI: `--model {ndt2,ibl_mtm,neuroformer} --pred_window 0.25/0.5/1.0`
   - 完整 training loop: DataLoader → train → periodic eval → save best model
   - 输出：best_model.pt + results.json 到 `results/logs/phase1_benchmark_{model}_{window}ms/`

6. **`neural-benchmark/run_all_benchmarks.sh`**：
   - 批量训练脚本：3 模型 × 3 窗口 = 9 个实验

7. **`neural-benchmark/visualize_benchmarks.py`**（约 280 行）：
   - 对比可视化脚本（4 张图：fp-bps 柱状图、per-bin 衰减、R² 柱状图、综合表+雷达图）

**关键 Bug 修复**：

1. **Spike binning 时间戳 bug**（严重）：
   - 问题：`bin_spike_events(ts, ui, n_units, start, end, ...)` 中 start 使用了绝对时间（如 97.627s），但 `torch_brain.Dataset.get()` 返回的 spike timestamps 是相对时间（0 到 window_length）
   - 症状：预测窗口 spike counts 全为 0（所有 spikes 被 clip 到 bin 0）
   - 修复：使用 `data.start`（=0）和 `data.end`（=window_length）作为 binning 参考

2. **IBL-MtM / Neuroformer 数据泄漏 bug**（严重）：
   - 问题：causal mask 允许当前位置 attend 到自身，而预测窗口位置的输入包含了 target spike counts
   - 症状：IBL-MtM 2 epochs 即达 fp-bps=2.15（正常值应 < 0.2），R²=0.90
   - 修复：在 forward() 和 compute_loss() 中将预测窗口输入置零（与 NDT2 的 MAE 方式一致）
   - 修复后：IBL-MtM 2 epochs fp-bps=0.11（合理）

**Sanity Check 验证结果**（2 epochs, pred_window=250ms）：

| 模型 | 参数量 | train_loss | val_nll | fp-bps | R² |
|------|--------|-----------|---------|--------|-----|
| NDT2 | 4.8M | 0.3251 | 0.3186 | 0.105 | 0.220 |
| IBL-MtM | 10.7M | 0.3252 | 0.3179 | 0.112 | 0.221 |
| Neuroformer | 4.9M | 0.3436 | 0.3266 | 0.019 | 0.192 |

**20 epochs 验证结果**（NDT2, pred_window=250ms）：
- fp-bps=0.140, R²=0.232（指标正常提升中）
