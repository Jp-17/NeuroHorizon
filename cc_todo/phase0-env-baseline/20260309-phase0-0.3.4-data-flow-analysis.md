### 关键发现

#### 1. 时间尺度属性层次关系

```
domain (663.2s, 整个录制)
 ├── trials (193 个, 含 valid/invalid)
 │    └── valid trials (157 个)
 │         ├── hold_period (target_on → go_cue, 0.247s avg)
 │         ├── reach_period (go_cue → stop, 1.022s avg)
 │         └── return_period (stop → trial_end, 2.013s avg)
 ├── random_period = domain - (所有 movement phases + invalid trials)
 ├── cursor_outlier_segments (检测到的行为异常区段)
 └── Split:
      ├── train_domain = domain - (valid_trials ∪ test_trials).dilate(1.0)
      ├── valid_domain = 10% valid trials
      └── test_domain = 20% valid trials
```

#### 2. Trial 边界不可知

- **采样是纯时间域的**：`get_sampling_intervals()` 返回 `{split}_domain` 属性，本质是连续时间段
- **窗口可跨越 trial 边界**：一个 1.0s 训练窗口可能包含一个 trial 的 return_period 尾部 + 下一个 trial 的 hold_period 开头
- **模型不显式感知 trial start/end**：不存在 trial boundary token 或 trial-level attention mask

#### 3. Split 创建中的 dilate(1.0) 安全间距

- valid/test trials 在从 domain 中差集时被 dilate 1.0s（前后各扩展 1s）
- 这确保 train_domain 中的窗口不会包含紧邻 valid/test trial 的数据，防止数据泄漏
- train_domain 因此有 38 个不连续的 intervals（被 valid/test trials + 安全间距分割）

### A4. 为什么 hold_period 时长不固定？

**神经科学设计原理（Variable Delay Paradigm）：**

hold_period 时长刻意设计为随机变化（本 session 约 0.2–0.5s，均值 0.247s），而非固定值。核心原因：

1. **防止动作预期（Anti-anticipation）**：若 hold 时长固定，猴子会在 hold 快结束时开始"预期性"的神经准备甚至肌肉激活，使 go cue 前的神经活动包含运动准备信号，导致 hold_period 神经状态"被污染"，无法研究纯粹的持续注意 / 目标保持状态。

2. **分离神经状态（Deconfounding）**：变长 hold 使研究者可以对 go cue 前不同时刻的神经活动进行对比，分离"静息 → 目标出现 → 运动准备"各阶段的神经动力学。这是运动皮层研究的经典实验设计（Churchland et al., 2012 Science；Yu et al., 2021 Neuron）。

3. **对 NeuroHorizon 的影响**：
   - 模型训练窗口可能跨越不同时长的 hold 段，带来天然的时长多样性
   - 250ms 固定窗口在 hold_period 期间只能看到一段速度接近 0 的"安静"神经活动
   - 这使 hold_period 下的神经表征成为很好的"基线对照"条件，有助于评估模型的静态编码能力

### A5. 用本数据集能否计算 PSTH？经典做法是什么？

**结论：可以，且是经典标准操作。**

**PSTH（Peri-Stimulus Time Histogram）**：
- 以某一事件为对齐点（go cue = `trials.go_cue_time`，与 `reach_period.start` 完全相同），在该事件前后一段时间窗口内，对每个神经元的发放率进行跨 trial 平均
- 本数据集 157 个有效 trial（c_20131003），每 trial 都有明确的 go cue 时间，完全符合 PSTH 计算前提

**可用字段（HDF5 实际验证）**：
| 字段 | 路径 | 说明 |
|------|------|------|
| go cue 时间 | `trials.go_cue_time`（= `reach_period.start`）| 对齐事件 |
| 方向 ID | `trials.target_id` | 整数 0–7，8个方向 |
| 方向角度 | `trials.target_dir` | 弧度，0=右，π/2=上，π=左，-π/2=下 |
| 有效 trial | `trials.is_valid` | bool，157/193 有效 |

**经典做法（center-out reaching 任务）**：
1. **按目标方向分组**：`target_id` 0–7 对应 8 个方向；PSTH 按方向分组计算，可得"方向调谐"PSTH 曲线
2. **对齐事件**：对齐 go cue（`reach_period.start`），时间窗口取 [-300ms, +800ms]
3. **bin size**：20ms bin + Gaussian 平滑（σ=1 bin ≈ 20ms）
4. **平均**：对同方向所有 trial 的发放率取均值，得每方向的平均 PSTH；preferred direction 上峰值更高

**实际实现（c_20131003 session）**：
- 本次已在 `explore_brainsets.py` 末尾实现并生成 `03_direction_tuning_psth.png`
- 各方向 trial 数：约 15–22 trial（均匀分布，最小 15，最大 22）
- 71 个神经元中 top-5 峰值发放率：116 Hz / 73 Hz / 62 Hz / 59 Hz / 54 Hz
- Population PSTH 在 go cue 后约 100–200ms 有明显峰值，与 reaching 运动启动一致

**与本项目的关系**：
- PSTH 可用于：(a) 验证神经元质量，(b) 可视化模型预测 vs 真实发放率，(c) Phase 1 定性评估工具
- 方向调谐 PSTH 展示 M1 神经元的经典调谐特性，有助于直觉性验证数据质量

## Part B: 训练与评估中的数据流

### 训练流程

1. **采样**：`RandomFixedWindowSampler`
   - 从 train_domain (38 intervals, 433.4s) 生成固定长度窗口 (1.0s)
   - 每 interval: N = floor(interval_length / window_length) 个窗口
   - 随机偏移 + 全局 shuffle
   - 本 session 每 epoch 约 417 个训练窗口

2. **损失权重**：`resolve_weights_based_on_interval_membership()`
   - 每个 cursor timestamp 根据所属 movement phase 获得权重
   - reach_period: 5.0x（模型重点学习 reach 期间的运动解码）
   - hold_period: 0.1x（holding 状态变化少，权重降低）
   - return_period: 1.0x, random_period: 1.0x
   - cursor_outlier_segments: 0.0（完全排除异常行为区段）
   - 权重是乘法叠加的（若 timestamp 同属 reach_period 和 outlier_segments，最终权重 = 5.0 * 0.0 = 0.0）

3. **损失计算**：
   - `loss = MSE(pred, target) * weight`，按 timestamp 加权
   - 总损失 = sum(taskwise_loss * n_seq_with_task) / batch_size

### 评估流程

1. **采样**：`DistributedStitchingFixedWindowSampler`
   - 从 valid_domain (16 intervals, 49.8s) 生成重叠窗口
   - window=1.0s, step=0.5s（50% 重叠）
   - 本 session 约 76 个评估窗口
   - 重叠窗口被分配到同一 GPU rank，便于本地 stitching

2. **eval_mask 机制**：
   - YAML 配置: `eval_interval: movement_phases.reach_period`
   - 意图：仅在 reach_period 时间戳上计算 R²
   - **发现**：`multitask_readout.py:237` 中读取 `data.config.get("eval_interval")`，从顶层 config 字典读取
   - 但 YAML 中 `eval_interval` 嵌套在 `multitask_readout` 列表项内，不在顶层
   - 因此 `data.config.get("eval_interval")` 返回 None，eval_mask 默认全为 True
   - **实际效果**：R² 在所有 cursor timestamps 上计算，而非仅 reach_period

3. **Stitching 聚合**：
   - 收集每个 session 的所有 window predictions
   - 重叠 timestamps 通过 `stitch()` 函数 mean-pool 合并
   - 每个 session 计算一个 R²，最后跨 session 取平均

## Part C: 批次数据维度分析（补充）

### C1. 1 个 Batch 是否包含多个 Session 的数据？

**结论：在多 session 训练配置下，是的。**

POYO+ 的 DataLoader 采用 `ConcatDataset`（多个 session 的 `IrregularTimeSeries` 数据集合并），将所有 session 的采样窗口混合后按 `batch_size` 组批。一个 batch（例如 `batch_size=8`）中可能包含来自不同 session 的样本。

不同 session 的关键差异：
- **神经元数目（units per session）**：Perich-Miller 数据集各 session 差异显著（约 40–100 个有效单元，c_20131003 有 71 个）
- **神经元 ID（unit IDs）**：跨 session 神经元不对应，同一位置的 unit_id 在不同 session 中代表不同神经元

### C2. 不同 Session 神经元数目不同时，输入/输出如何处理？

**POYO 的核心设计：token-based（非 neuron-matrix）表示**

**输入侧（spike tokens）**：
- 每个 spike 事件表示为一个 token：`(timestamp, unit_embedding)`
- unit_embedding 来自 `InfiniteVocabEmbedding`，将 unit_id 映射到 d_model 维向量
- 不同 session 的 unit_id 各自独立（通过 session-specific 命名空间区分）
- 同一 batch 中不同样本的 spike token 数量不同（稀疏 spike → 变长序列）
- **批次内处理**：使用 padding + attention mask（xformers 支持变长 attention），或统一截断到 `max_num_tokens`

**输出侧（query tokens）**：
- 每个输出查询 token 代表一对 `(time_bin, unit_id)`
- 不同 session 的 unit 数不同 → 不同样本的 query token 数不同
- 同样通过 padding / variable-length attention 处理

**Latent 瓶颈（Perceiver）**：
- latent array 大小固定（如 `num_latents=64`，与 neuron 数无关）
- 所有 spike token → cross-attention → 固定大小 latent → self-attention → 固定大小 latent → cross-attention → output queries
- 这是 POYO 能处理可变神经元数的关键：瓶颈层固定，输入/输出层变长

### C3. 实际 Batch 数据维度

当前配置（小模型，250ms 窗口，Perich-Miller 单 session c_20131003）下的维度：

| 维度 | 值 | 说明 |
|------|-----|------|
| batch_size | 8 | 每批 8 个时间窗口 |
| 输入 spike token 数/样本 | 变长 ≈ 30–200 | 250ms × ~71 units × ~0.3 spk/bin，高度变化 |
| 输出 query token 数/样本 | 变长 = num_bins × num_units | 250ms/20ms × units_in_session |
| num_bins (output) | 12 | 250ms / 20ms = 12.5，取 12 |
| units/session | 71 (c_20131003) | 因 session 而异，约 40–100 |
| 输出 query 总数/样本 | 12 × 71 = 852 | 因 session 而异 |
| latent 大小 | 64 × d_model (128) | 固定，与 neuron 数无关 |
| d_model | 128 (small config) | 可调 |

**跨 session batch 时**：每个样本的 query 数 / spike token 数各自不同，batch 内不做统一维度对齐（xformers 支持 per-sample attention），从 DataLoader 角度是 list-of-variable-length-tensors 而非统一 3D tensor。
