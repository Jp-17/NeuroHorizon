### Part C: NLB 指标对齐

| 指标 | 全称 | 实现难度 | 与 NeuroHorizon 的关系 |
|------|------|----------|----------------------|
| **co-bps** | Co-smoothing Bits Per Spike | 高 | 完全不同的任务（预测 held-out neurons） |
| **fp-bps** | Forward Prediction Bits Per Spike | 中-高 | 与 AR 预测最接近，需适配评估格式 |
| **PSTH R²** | Trial-averaged R² | 低-中 | 已有 R² 基础，需加 trial-averaging |

**co-bps 计算方式**：
```
bps = (log_likelihood_model - log_likelihood_null) / (n_spikes * ln(2))
null model = neuron-specific mean firing rate
```

**推荐路径**：
1. ✅ **行为解码 R²**（NLB eval intervals 上）— 已支持，直接可用
2. 🔶 **fp-bps** — 与 NeuroHorizon AR 任务最接近，中等工作量
3. 🔴 **co-bps** — 需要重大 pipeline 改动，仅在竞争优势明确时实现

## 0.4.2 对比模型可行性分析

### NDT2 (Neural Data Transformer 2)

- **仓库**：[github.com/joel99/context_general_bci](https://github.com/joel99/context_general_bci)（MIT License）
- **论文**：NeurIPS 2023
- **输入格式**：20ms binned spike counts，spatiotemporal patches
- **预训练方式**：Masked Autoencoding (MAE)，50% masking ratio
- **NLB 测试**：有 NLB loaders，但单 context 未超过 NLB SoTA
- **运动皮层经验**：✅ 有（RTT, FALCON benchmark）
- **预训练权重**：❌ 未发布
- **适配难度**：**低-中** ⭐ **推荐**
  - 已有 NLB 数据加载器
  - 需编写 brainsets → binned counts 的转换
  - 需注册 ExperimentalTaskLoader
- **后续**：NDT3（2025）基于 NDT2 扩展到 100M+ 参数，也值得关注

#### Forward Prediction 原始实现分析

**是否在原实现中进行过 forward 推理预测**：✅ **是**（NLB fp-bps 是其核心评估任务之一）

**Observation window & Forward prediction window（NLB MC_Maze 标准定义）**：
- 数据集：NLB MC_Maze（jenkins_maze session，单 session）
- **Observation window**：约 **500ms**（NLB eval interval 700ms 减去末尾 200ms fp 部分）
- **Forward prediction window**：**200ms**（NLB eval interval 末尾 200ms，为 NLB 挑战标准定义）
- Bin 数：5ms bin → obs = 100 bins, fp = 40 bins；20ms bin → obs = 25 bins, fp = 10 bins

**实现方式（NDT2 的 MAE 适配）**：

NDT2 预训练用 MAE（随机 mask 50% tokens 然后重建），但推理时通过**改变 mask 模式**适配不同任务——本质上是 **"mask 什么就预测什么"**：

**NLB fp-bps 任务定义**：
- 每个 trial 分为 **observation window**（T bins，如 MC_Maze_Large 140 bins = 700ms）和 **forward prediction window**（T_fp bins，如 40 bins = 200ms）
- 模型输入：observation window 内的 held-in neurons 的 spike counts
- 模型输出：forward prediction window 内**全部 neurons（held-in + held-out）**的 predicted firing rates
- 评估公式：`bps = (NLL_null - NLL_model) / (n_spikes * ln(2))`，null model 为 per-neuron mean firing rate

**NDT2 推理流程**：
1. observation window 的 held-in neuron tokens 正常输入 encoder
2. forward prediction window 的所有位置插入 **mask tokens**（或 zero-valued tokens）——模型在这些位置**看不到任何真实 spike 数据**
3. encoder 处理可见 tokens，decoder 在 masked positions 重建 firing rates
4. decoder 输出即为 forward prediction 的预测值

**与 NeuroHorizon 的关键区别**：
| | NDT2 (MAE) | NeuroHorizon (AR) |
|---|---|---|
| 预测方式 | **并行填充**：一次性重建所有 masked future bins | **自回归逐步**：逐 bin 预测 |
| 训练目标 | 随机 mask 重建（通用，不特化 fp） | 直接优化 causal prediction |
| 推理时信息流 | future bins 之间可互相 attend（bidirectional decoder） | future bins 只能看到过去（causal mask） |
| fp 性能 | 合理但非最优（random mask ≠ causal mask） | 直接对齐 fp 任务 |

**MtM 论文（Zhang et al. 2024）的消融实验证实**：
- NDT2 默认 random token masking → fp-bps ≈ 0.50
- 专用 causal masking → fp-bps ≈ 0.52（更好）
- Multi-task Masking（含 causal）→ fp-bps ≈ 0.54（最佳）

这说明 MAE 的 random masking 对 fp 任务并非最优——NeuroHorizon 的 causal decoder 在 fp 任务上有**架构层面的天然优势**。

**其他模型的 fp 实现对比**：
- **LFADS/AutoLFADS**：RNN 前向运行，future bins 无输入，靠学到的 autonomous dynamics 外推
- **GPFA**：Gaussian Process temporal prior 自然外推到未来时间点
- **STNDT**：与 NDT2 相同，mask future bins 然后重建

### Neuroformer

- **仓库**：[github.com/a-antoniades/Neuroformer](https://github.com/a-antoniades/Neuroformer)（MIT License）
- **论文**：ICLR 2024
- **输入格式**：Neuron ID + time interval tokens（自回归 token 化）
- **预训练方式**：GPT-style autoregressive + 多模态对比学习
- **NLB 测试**：❌ 无（仅在 two-photon calcium imaging 上测试）
- **运动皮层经验**：❌ 无
- **预训练权重**：❌ 未发布
- **适配难度**：**高** ⚠️ **不推荐用于运动皮层**
  - 设计面向 calcium imaging，非电生理
  - token 化方式与 binned counts 不同
  - 无运动皮层基准可参考

#### Forward Prediction 原始实现分析

**是否在原实现中进行过 forward 推理预测**：✅ **是**（架构原生支持，是其核心训练目标）

**实现方式**：
- Neuroformer 是**真正的自回归生成模型**（GPT-style 因果 Transformer），与 NDT2 的 MAE 有本质区别
- **Token 粒度**：每个 token 是一个 **spike event**（两步预测：先预测触发的 neuron ID，再预测该 spike 在时间 bin 内的偏移量），而非 binned population counts
- **时间窗口组织**：将 spike train 划分为 **context window (w_c)**（observation）和 **prediction window (w_p)**（forward）
- **训练目标**：next-token prediction——在 context tokens + 已生成 spike tokens 条件下，逐步预测下一个 spike 的 neuron ID 和时间间隔
- **推理时**：将 observation window 内的 spike token 序列送入模型，自回归逐步生成 forward window 内的 spike tokens（每步先生成 neuron ID，再生成 time interval），直到生成 end-of-state marker
- **多模态版本**：视觉帧特征 tokens 拼接进 context，实现视觉条件下的 spike 生成

**Observation window & Forward prediction window（论文设置）**：
- 数据集：mouse V1 calcium imaging（Stringer et al. 2019）、mouse hippocampus（Grosmark et al.）
- **Context window (w_c)**：可配置超参数，论文中具体时长未统一披露；代码中默认约 **0.5–1s** 的 spike token 序列
- **Prediction window (w_p)**：可配置，**开放式生成**——自回归产生 spike tokens 直到填满 prediction window，无固定上限
- ⚠️ **论文未在 NLB MC_Maze 或运动皮层电生理数据上评估 forward prediction**，无可与 NLB fp-bps 标准直接对比的 obs/fp 窗口数值

**评估指标**：
- Population Vector Correlation（PVC）：预测 spike train 与真实 spike 序列的相关性
- PSTH 相关性；行为解码 R²（multi-modal 条件生成实验）
- **未使用 NLB fp-bps 标准指标**

**与 NeuroHorizon 的比较**：

| 维度 | Neuroformer | NeuroHorizon |
|------|-------------|--------------|
| 预测方式 | 逐 **spike** 自回归（spike-level tokens） | 逐 **bin** 自回归（20ms bin-level tokens） |
| 自回归步数 | = 总 spike 数（高放电率时步数极多） | = bin 数（固定：12/25/50 steps） |
| 推理效率 | 高放电率时极慢（每步仅预测 1 spike） | 固定步数，适合 Neuropixels 密集数据 |
| 验证数据 | 钙成像（V1、海马），**从未测试电生理运动皮层** | 目标：Neuropixels 运动皮层（Perich-Miller） |
| Forward 能力 | 架构原生，开放式生成 | 架构原生，固定 forward window |


### NDT-MtM (Multi-task Masking)

- **仓库**：[github.com/colehurwitz/IBL_MtM_model](https://github.com/colehurwitz/IBL_MtM_model)（MIT License）
- **论文**：NeurIPS 2024
- **输入格式**：20ms binned spike counts，2s trials
- **预训练方式**：4 种 masking 策略（temporal / neuron / intra-region / inter-region）
- **NLB 测试**：❌ 无（仅 IBL repeated-site）
- **运动皮层经验**：❌ 无
- **预训练权重**：✅ HuggingFace（需组织成员权限）
- **适配难度**：**中-高**
  - 紧耦合 IBL/Neuropixels 数据格式
  - 多 region masking 对单 area 录制不太相关
  - 预训练权重 domain 不匹配（visual/hippocampal → motor cortex）

#### Forward Prediction 原始实现分析

**是否在原实现中进行过 forward 推理预测**：✅ **是**（通过 4 种 masking 任务之一的 temporal causal masking，是其核心评估维度）

**实现方式**：
- MtM 框架包含 **4 种 masking 任务**：temporal（时间 mask）/ neuron（神经元 mask）/ intra-region（同脑区 mask）/ inter-region（跨脑区 mask）
- **Temporal causal masking**（对应 forward prediction）：将时间窗口**末尾 K 个 time bins 全部 mask**，模型需重建这些未来时间步的 firing rates
- 推理方式：**MAE 并行预测**——一次性重建所有 masked future bins（**非逐步自回归**）
- 模型通过可学习的 **"task prompt" token** 区分当前 masking 任务，推理时只需切换 prompt token 即可执行不同类型的预测（forward / neuron / region）
- 4 种 masking 任务通过 multi-task learning 联合训练，推理按需切换

**Observation window & Forward prediction window（论文 IBL 设置）**：
- 数据集：IBL repeated-site Neuropixels（48 animals，26,736 neurons，多脑区；非运动皮层）
- **Time bin**：20ms
- **时间窗口长度**：约 **1.5–2s**（75–100 time bins）
- **Forward prediction window**：约 **200ms**（≈10 bins，即时间窗口末尾约 10–15%）
- **Observation window**：约 **1.3–1.8s**（时间窗口前缀部分，65–90 bins）

**fp-bps 结果**（论文报告，IBL repeated-site sessions）：

| 配置 | fp-bps |
|------|--------|
| NDT2 baseline（random masking） | ≈ 0.50 |
| NDT2 + 专用 causal masking | ≈ 0.52（+4%） |
| NDT2 + 4-task MtM（含 causal）| **≈ 0.54（+8%，最佳）** |

**与 NeuroHorizon 的比较**：

| 维度 | NDT-MtM | NeuroHorizon |
|------|---------|--------------|
| 预测方式 | **并行预测**（MAE 风格，一次性填充 future bins） | **逐步自回归**（causal mask，step-by-step） |
| Future bins 间的信息流 | Bidirectional（future bins 之间可互 attend） | Causal（future bin 只能看到更早的 bins） |
| 训练目标 | 4 任务联合，causal masking 仅占 1/4 | 专门针对 AR forward prediction 设计 |
| fp 的架构支持 | 通过 masking 策略适配，非原生 causal | 原生 causal decoder（架构层面保证因果性） |
| 数据域 | IBL（视觉/海马，多脑区） | 目标：运动皮层（Perich-Miller） |

**关键结论**：MtM 的 fp 提升证明了 causal masking 的价值，但其**并行预测**的信息流（future bins 间可互 attend）与 NeuroHorizon 的**严格因果**自回归存在架构层面的本质差异——MtM 的 fp 是"近似 causal"而非严格因果。


### 综合评估

| 模型 | 适配难度 | 关键优势 | 关键风险 | 推荐度 |
|------|----------|----------|----------|--------|
| **NDT2** | 低-中 | 成熟的运动皮层代码库；有 NLB loaders | 无预训练权重；单 context 未超 NLB SoTA | ⭐⭐⭐ |

**结论**：**NDT2 是最强候选**。它是唯一在运动皮层数据上验证过的模型，有现成的 NLB 基础设施，且提供了清晰的新数据集接入点。后续版本 NDT3（2025, 100M+ params）也值得关注。

## 0.4.3 基于 torch_brain 统一数据接口的对比方案（可行性评估）

### 核心思路

**不改造各模型的内部数据 pipeline，而是让 torch_brain 提供统一的"数据窗口合约"**：

```
torch_brain (统一数据层)
  ├── 提供：train/valid/test 时间区间 (intervals)
  ├── 提供：每个区间内的原始 spike events (timestamps + unit_index)
  ├── 提供：行为数据 (hand.vel, hand.pos 等)
  └── 提供：unit metadata (unit_number, count 等)
       │
       ▼
各模型自行适配（薄适配层）
  ├── NDT2:     spike events → 20ms binned counts → spatiotemporal patches
  ├── Neuroformer: spike events → neuron ID + dt tokens
  ├── NDT-MtM:  spike events → 20ms binned counts → IBL-style format
  └── NeuroHorizon: spike events → POYO tokenization (native)
```

### 可行性分析

**torch_brain 已有的统一接口**：

1. **`get_sampling_intervals(split)`** — 返回 `{recording_id: Interval}` 字典，定义 train/valid/test 的时间区间
   - Perich-Miller: `train_domain` / `valid_domain` / `test_domain`
   - NLB: 同上，且与 NLB 原始 split 完全一致（已验证）
2. **HDF5 统一结构** — `spikes.timestamps` + `spikes.unit_index`（IrregularTimeSeries），所有数据集统一
3. **行为数据** — `hand.timestamps` + `hand.vel` / `hand.pos`
4. **Domain intervals** — 明确的时间边界，支持精确的窗口裁剪

**每个模型需要的适配层**（估算工作量）：

| 模型 | 适配层核心逻辑 | 工作量 |
|------|---------------|--------|
| **NDT2** | 读取 intervals → 切出 spike events → `np.histogram` binning (20ms) → reshape 为 (neurons, time_bins) | **约 100 行** Python，1-2 天 |
| **Neuroformer** | 读取 intervals → 切出 spike events → 转为 `{neuron_id, dt}` token 序列 | **约 150 行**，2-3 天（需理解其 tokenizer） |
| **NDT-MtM** | 读取 intervals → 切出 spike events → binning (20ms) → 对齐 IBL session dict 格式 | **约 120 行**，2 天 |
| **NeuroHorizon** | 原生支持，无需适配 | **0** |

**评估统一化**（关键）：
- 所有模型在**完全相同的 test intervals** 上评估 → 保证绝对公平
- 评估指标统一计算：R²（行为解码）/ Poisson NLL / PSTH correlation
- 不依赖各模型自带的评估代码，而是在 torch_brain 层面统一实现

### 具体实现方案

```python
# 薄适配层伪代码（以 NDT2 为例）
class TorchBrainToNDT2Adapter:
    def __init__(self, dataset, split="train", bin_size_s=0.02):
        self.intervals = dataset.get_sampling_intervals(split)
        self.bin_size = bin_size_s

    def get_trial(self, recording_id, interval_idx):
        """从 torch_brain 数据中切出一个 trial，转为 NDT2 格式"""
        recording = dataset.get_recording(recording_id)
        start, end = self.intervals[recording_id][interval_idx]

        # 切出时间窗口内的 spike events
        mask = (recording.spikes.timestamps >= start) & (recording.spikes.timestamps < end)
        spike_times = recording.spikes.timestamps[mask]
        spike_units = recording.spikes.unit_index[mask]

        # binning: spike events → (n_units, n_bins) 矩阵
        n_bins = int((end - start) / self.bin_size)
        n_units = len(recording.units.id)
        binned = np.zeros((n_units, n_bins))
        for t, u in zip(spike_times, spike_units):
            bin_idx = min(int((t - start) / self.bin_size), n_bins - 1)
            binned[u, bin_idx] += 1

        return binned  # NDT2 可直接使用
```

### 难度重新评估（基于统一数据接口方案）

| 模型 | 原评估 | 修正后评估 | 降低原因 |
|------|--------|-----------|---------|
| **NDT2** | 低-中 | **低** | 只需 binning 适配层 + 模型配置调整，无需重写数据 pipeline |
| **Neuroformer** | 高 | **中** | tokenization 需理解但数据获取统一了；不推荐的原因仍在（未在运动皮层验证） |
| **NDT-MtM** | 中-高 | **中** | 数据格式统一后只需对齐 session dict；multi-region masking 仍不太相关 |

### 结论

✅ **完全可行**。torch_brain 的 `get_sampling_intervals` + HDF5 spike events 结构天然构成了统一数据合约：

1. **数据公平性**：所有模型使用完全相同的 train/valid/test 时间区间和原始 spike 数据
2. **评估公平性**：在相同 test intervals 上用相同指标评估
3. **适配复杂度低**：每个模型只需一个 ~100-150 行的薄适配层（spike events → 各自格式）
4. **不侵入各模型代码**：适配层是外部 wrapper，各模型的 model/training/inference 代码不需要修改

**推荐实施顺序**：
1. 先实现 NDT2 适配层（工作量最小，验证方案可行性）
2. 在 Perich-Miller 上完成 NDT2 vs NeuroHorizon 对比
3. 视需要扩展到 NDT-MtM / Neuroformer
