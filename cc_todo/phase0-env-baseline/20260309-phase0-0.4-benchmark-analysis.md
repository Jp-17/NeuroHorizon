| 属性 | Train File | Test File |
|------|-----------|-----------|
| Trials | 100 | 100 |

| Spikes | 131,669 | 32,897 |

**Brainsets 划分**：
- train_domain: 60 trials

**与 Perich-Miller 的对比**：
- NLB MC_Maze 是单 session（vs Perich-Miller 多 session 多动物）
- NLB 有明确的 held-in/held-out neuron 设计（vs Perich-Miller 无此概念）
- NLB 数据有 eye tracking（vs Perich-Miller 仅 cursor）
- NLB trial 结构更标准化（固定 0.7s eval interval）

### Part B: NLB 数据适配性调查

#### B1: Split 一致性检查

| 划分方式 | train | valid/val | test |
|----------|-------|-----------|------|
| NLB 原始 (train_mask_nwb) | 75 | — | 25 |
| Brainsets (train/valid/test_domain) | 60 | 15 | 25 |
| split_indicator 字段 | 'train' | 'val' | — |

**Cross-mapping 结果**：
- brainsets train_domain (60 trials) → 全部来自 NLB train (60/75)
- brainsets valid_domain (15 trials) → 全部来自 NLB train (15/75)
- brainsets test_domain (25 trials) → 全部来自 NLB test (25/25)

**结论**：✅ **Split 完全一致**。Brainsets 将 NLB 的 75 个 train trials 进一步拆分为 60 train + 15 valid，test set 完全匹配。不存在数据泄露问题。

#### B2: Held-in/held-out 机制

- **Held-in neurons**: 107 units（出现在 train 和 test 文件中）
- **Held-out neurons**: 35 units（仅出现在 train 文件中）
- 匹配通过 `unit_number` 而非 `unit_id`（string ID 因电极重映射不同）

**对 NeuroHorizon 评估的影响**：
- NeuroHorizon 当前训练使用**全部 142 个 neurons**，评估行为解码 R²
- NLB 的核心任务 co-bps 要求**仅用 107 个 held-in neurons 预测 35 个 held-out neurons 的发放率**
- 这是**完全不同的任务**——直接使用 NLB 排行榜数值对比不合适
- 但行为解码 R² 在 NLB eval intervals 上仍可与其他模型横向比较

#### B3: nlb_eval_intervals

- 100 个评估区间（每个 trial 一个），固定 0.7s 时长
- train/valid/test 分配遵循 brainsets split (60/15/25)
- POYO NLB config 已正确配置 `eval_interval: nlb_eval_intervals`

#### B4: 其他 NLB 子数据集

| 子数据集 | 适配难度 | 备注 |
|----------|----------|------|
| MC_RTT | 中 | 类似 spike 格式，不同任务结构 |
| Area2_Bump | 中-高 | 不同脑区（S2），不同任务 |
| DMFC_RSG | 高 | 完全不同的任务范式（timing-based） |
| MC_Cycle | 中 | 类似脑区但周期性运动 |

当前 brainsets 仅包含 MC_Maze pipeline，其他子数据集需额外实现。

### Part C: NLB 指标对齐

| 指标 | 全称 | 实现难度 | 与 NeuroHorizon 的关系 |
|------|------|----------|----------------------|
| **co-bps** | Co-smoothing Bits Per Spike | 高 | 完全不同的任务（预测 held-out neurons） |
| **fp-bps** | Forward Prediction Bits Per Spike | 中-高 | 与 AR 预测最接近，需适配评估格式 |
| **PSTH R²** | Trial-averaged R² | 低-中 | 已有 R² 基础，需加 trial-averaging |

**co-bps 计算方式**：
```
bps = (log_likelihood_model - log_likelihood_null) / (n_spikes * ln(2))
null model = neuron-specific mean firing rate
```

**推荐路径**：
1. ✅ **行为解码 R²**（NLB eval intervals 上）— 已支持，直接可用
2. 🔶 **fp-bps** — 与 NeuroHorizon AR 任务最接近，中等工作量
3. 🔴 **co-bps** — 需要重大 pipeline 改动，仅在竞争优势明确时实现

**nlb_tools 状态**：未安装。安装命令：`pip install nlb-tools`

## 0.4.2 对比模型可行性分析

### NDT2 (Neural Data Transformer 2)

- **仓库**：[github.com/joel99/context_general_bci](https://github.com/joel99/context_general_bci)（MIT License）
- **论文**：NeurIPS 2023
- **输入格式**：20ms binned spike counts，spatiotemporal patches
- **预训练方式**：Masked Autoencoding (MAE)，50% masking ratio
- **NLB 测试**：有 NLB loaders，但单 context 未超过 NLB SoTA
- **运动皮层经验**：✅ 有（RTT, FALCON benchmark）
- **预训练权重**：❌ 未发布
- **适配难度**：**低-中** ⭐ **推荐**
  - 已有 NLB 数据加载器
  - 需编写 brainsets → binned counts 的转换
  - 需注册 ExperimentalTaskLoader
- **后续**：NDT3（2025）基于 NDT2 扩展到 100M+ 参数，也值得关注

### Neuroformer

- **仓库**：[github.com/a-antoniades/Neuroformer](https://github.com/a-antoniades/Neuroformer)（MIT License）
- **论文**：ICLR 2024
- **输入格式**：Neuron ID + time interval tokens（自回归 token 化）
- **预训练方式**：GPT-style autoregressive + 多模态对比学习
- **NLB 测试**：❌ 无（仅在 two-photon calcium imaging 上测试）
- **运动皮层经验**：❌ 无
- **预训练权重**：❌ 未发布
- **适配难度**：**高** ⚠️ **不推荐用于运动皮层**
  - 设计面向 calcium imaging，非电生理
  - token 化方式与 binned counts 不同
  - 无运动皮层基准可参考

### NDT-MtM (Multi-task Masking)

- **仓库**：[github.com/colehurwitz/IBL_MtM_model](https://github.com/colehurwitz/IBL_MtM_model)（MIT License）
- **论文**：NeurIPS 2024
- **输入格式**：20ms binned spike counts，2s trials
- **预训练方式**：4 种 masking 策略（temporal / neuron / intra-region / inter-region）
- **NLB 测试**：❌ 无（仅 IBL repeated-site）
- **运动皮层经验**：❌ 无
- **预训练权重**：✅ HuggingFace（需组织成员权限）
- **适配难度**：**中-高**
  - 紧耦合 IBL/Neuropixels 数据格式
  - 多 region masking 对单 area 录制不太相关
  - 预训练权重 domain 不匹配（visual/hippocampal → motor cortex）

### 综合评估

| 模型 | 适配难度 | 关键优势 | 关键风险 | 推荐度 |
|------|----------|----------|----------|--------|
| **NDT2** | 低-中 | 成熟的运动皮层代码库；有 NLB loaders | 无预训练权重；单 context 未超 NLB SoTA | ⭐⭐⭐ |
| **Neuroformer** | 高 | 新颖的自回归方法；优雅的多模态设计 | 从未在电生理/运动皮层测试 | ⭐ |
| **NDT-MtM** | 中-高 | 有预训练权重；原理性的多任务 masking | 紧耦合 IBL；多 region focus 不匹配 | ⭐⭐ |

**结论**：**NDT2 是最强候选**。它是唯一在运动皮层数据上验证过的模型，有现成的 NLB 基础设施，且提供了清晰的新数据集接入点。后续版本 NDT3（2025, 100M+ params）也值得关注。
